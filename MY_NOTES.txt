WHAT I BUILT
A PHP webhook that Shopify calls when an order is created. It reads the pickup
point info and updates the order with three things:
- Shipping line title: "Ackermans - EXPRESS - 1569"
- Metafields: delivery.method and delivery.branch_code
- Tags: delivery:express, branch:1569, click-and-collect-express


WHAT EACH FILE DOES

index.php
The URL Shopify hits. Validates the request then runs the pipeline.

src/Config.php
Reads .env and sets up all credentials.

src/Logger.php
Writes to logs/app.log.

src/ShopifyClient.php
Makes GraphQL calls to Shopify. Gets the pickup point externalId and shipping price.

src/PickupPointParser.php
Splits "Ackermans-EXPRESS-1569" into courier, method, branch. Builds the title and tags.

src/OrderUpdater.php
Runs the three mutations: shipping line, metafields, tags.


HOW TO START THE PROJECT

1. Open terminal in project folder
   cd /Users/shaqielheradien/Downloads/shopify-pickup-handler

2. Start PHP server - keep this tab open
   php -S localhost:8000

3. Open a second terminal tab and run ngrok
   ngrok http 8000

4. If ngrok gave you a new URL, update the webhook in Shopify
   Admin > Settings > Notifications > Webhooks > your webhook > update URL
   New URL format: https://xxxx.ngrok-free.app/index.php

5. Send a test from Shopify to confirm the endpoint is working
   Admin > Settings > Notifications > Webhooks > Send test notification

6. Check the log
   cat logs/app.log


HOW TO TEST THE FULL FLOW

Run this from the project folder. Replace the order ID with a real one.

php -r "
require 'src/Config.php';
require 'src/Logger.php';
require 'src/ShopifyClient.php';
require 'src/PickupPointParser.php';
require 'src/OrderUpdater.php';
\$orderId = 'gid://shopify/Order/YOUR_ORDER_ID';
\$parsed  = parse_external_id('Ackermans-EXPRESS-1569');
\$price   = get_shipping_price(\$orderId);
update_shipping_line(\$orderId, format_shipping_title(\$parsed), \$price['amount'], \$price['currencyCode']);
update_metafields(\$orderId, \$parsed['method'], \$parsed['branch_code']);
update_tags(\$orderId, build_tags(\$parsed));
echo 'Done' . PHP_EOL;
"

Then check the log. You should see three INFO lines: shipping line updated,
metafields set, tags added.


HOW TO VERIFY IN GRAPHIQL

Open GraphiQL in your Shopify store, set version to 2026-04, run this:

query {
  order(id: "gid://shopify/Order/YOUR_ORDER_ID") {
    shippingLines(first: 1) {
      nodes { code title }
    }
    metafields(first: 10, namespace: "delivery") {
      edges { node { key value } }
    }
    tags
  }
}

You should see title "Ackermans - EXPRESS - 1569", metafields EXPRESS and 1569,
and the three tags.


IF SOMETHING BREAKS

502 error from Shopify
PHP server is not running. Run php -S localhost:8000.

HMAC mismatch in logs
Webhook secret in .env is wrong. Go to Admin > Settings > Notifications >
scroll to bottom > click show next to signing secret > copy into .env.

Access denied for orderEditBegin
App token missing write_order_edits scope. Go to Shopify app > Configuration >
enable write_order_edits > save > reinstall app > copy new token into .env.

ngrok URL changed
Update the webhook URL in Shopify to match the new ngrok URL.


IMPORTANT

The externalId in index.php is hardcoded to "Ackermans-EXPRESS-1569" for testing.
In production with Pickup Point Generator enabled, uncomment get_pickup_external_id()
and remove the hardcoded line.

Never commit .env - it contains your real API token. It is already in .gitignore.
